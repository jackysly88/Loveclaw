# LoveClaw：八字缘分匹配

基于八字与偏好的同城缘分匹配技能。用户通过对话完成报名，系统每日跑批匹配，晚间查看或推送结果。支持同性匹配。

当前版本：**21.21.21**

## 功能概述

- **对话式报名**：按步骤收集手机号、资料与可选照片；结束前会询问是否开启「每日匹配结果推送」。
- **八字推算**：根据出生日期与时辰生成八字，用于匹配打分。
- **缘分匹配**：同城、双向偏好一致的前提下，按算法择优配对。
- **隐私保护**：对方联系方式仅在匹配关系建立后展示；每个用户只能查询自己的数据。
- **定时任务**：每日匹配（约 19:50）与晚间报告推送（约 20:00）。

## 安全架构

- **服务端运算**：匹配算法、八字计分、报告生成均在 FC 云函数内完成。Skill 端仅做用户交互和通知路由，不包含匹配逻辑，不接触全量用户数据。
- **用户数据隔离**：每个 API 请求通过 `X-Loveclaw-User` 头标识身份，FC 端校验后仅返回该用户自己的数据。报告获取使用 `/api/my-report`（用户级），不存在返回全量数据的端点。
- **单一凭证**：skill 安装只需 `LOVECLAW_API_TOKEN`，无管理员凭证，无云基础设施密钥。
- **通知路由**：报告投递优先使用 OpenClaw 官方会话路由，渠道目标按规范格式化（Feishu `user:ou_xxx`、WhatsApp E.164 等）。
- **图片安全**：私有 OSS 图片使用签名 URL 访问。

## 安装

```bash
# ClawHub
clawhub install loveclaw

# 手动
npm install
```

安装完成后不要修改 `.env` 中的 LoveClaw 配置。

## 唤醒词

**推荐**：`启动爱情龙虾`

## 命令一览

| 输入 | 说明 |
| --- | --- |
| `启动爱情龙虾` | 进入报名流程 |
| `我的档案` / `查看档案` | 查看个人资料与匹配历史 |
| `今日匹配` | 查看当日匹配结果 |
| `匹配记录` | 查看历史匹配记录 |
| `开启推送` / `关闭推送` | 管理每日推送 |
| `取消报名` | 删除档案与照片 |

## 报名流程

发送「启动爱情龙虾」后，按提示依次填写：手机号 → 姓名 → 性别 → 希望认识的性别（支持同性） → 出生日期 → 时辰 → 城市 → 照片（可跳过） → 是否开启每日推送 → 确认。

## 匹配规则

1. **同城**：双方城市一致。
2. **双向偏好**：性别偏好互相满足（含同性）。
3. **分数门槛**：双向八字匹配分均 ≥ 70。

## 每日节奏

- **19:50**：全员每日匹配。
- **20:00**：晚间报告推送（仅已开启推送的用户）。

## 数据存储

- **档案与匹配**：阿里云 TableStore（经云函数 API 读写）。
- **照片**：OSS 对象存储（签名 URL 访问）。

## 隐私说明

1. 手机号、照片等敏感信息仅用于匹配与通知。
2. `取消报名` 会清理档案与照片。
3. 匹配成功后向对方展示的联系方式以当前模板策略为准。
